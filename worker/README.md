# Воркер для парсинга книг с ast.ru

Асинхронный сборщик данных о книгах с сайта издательства АСТ.

## Возможности

- **Параллельная загрузка** с настраиваемой конкуррентностью (по умолчанию 30 потоков)
- **Ретраи с экспоненциальной задержкой** и джиттером
- **Ротация User-Agent** (реальные браузеры)
- **Кэш прогресса** — можно прервать и продолжить
- **Сохранение в JSONL** — по одной книге на строку
- **Лог ошибок** в отдельный файл
- **Пропуск рейт-лимитов** (HTTP 429/503) с увеличенной задержкой

## Собираемые поля

| Поле | Источник |
|------|----------|
| `title` | JSON-LD, og:title, заголовок `<h1>` |
| `author` | JSON-LD, books:author, характеристика «Автор» |
| `isbn` | JSON-LD (gtin13), books:isbn, характеристика «ISBN» |
| `description` | JSON-LD, og:description (SEO-описание) |
| `annotation` | Полный текст аннотации (таб «Описание») |
| `year` | Характеристика «Год издания», books:release_date |
| `pages` | Характеристика «Страниц» |
| `cover_type` | «Переплет», «Обложка» |
| `series` | Характеристика «Серия» |
| `cycle` | Характеристика «Цикл» |
| `thickness` | «Толщина (мм)» |
| `format` | «Ширина (мм)» + «Высота (мм)» |
| `publisher` | JSON-LD brand, «Издательство» |
| `weight` | «Вес (кг)» |
| `age_restriction` | «Знак информационной продукции» |
| `language` | «Язык» |
| `translator` | «Переводчик» |
| `illustrator` | «Художник», «Иллюстратор» |
| `editor` | «Редактор», «Составитель» |
| `category` | Хлебные крошки |
| `tags` | «Жанр», «Жанры» |
| `url` | URL страницы |

> **Не собираются (по запросу):** `фото` и `цена`.

Любые неизвестные поля сохраняются в `extras` отдельным словарём.


## Использование

### Базовый запуск

```bash
python3 -m worker --input /Users/rustamismagilov/Desktop/AST/sparseno/book_links1111.txt
```

### Параметры

```bash
python3 -m worker --input links.txt \
    --concurrency 50 \         # 50 одновременных запросов
    --output books.jsonl \    # куда сохранять
    --limit 100 \             # только первые 100 URL
    --offset 1000 \           # пропустить первые 1000
    --pretty \                # красивый JSON (каждый атрибут на новой строке)
    --verbose                 # подробный лог
```

### Доступные опции

- `-i, --input FILE` — файл со списком URL (обязательно)
- `-o, --output FILE` — выходной JSONL (по умолчанию `books.jsonl`)
- `--progress FILE` — файл прогресса (по умолчанию `books.progress.json`)
- `--errors FILE` — файл с ошибками (по умолчанию `books.errors.jsonl`)
- `-c, --concurrency N` — конкуррентность (по умолчанию 30)
- `--timeout SEC` — таймаут запроса (по умолчанию 60)
- `--max-retries N` — число ретраев (по умолчанию 4)
- `--limit N` — обработать только N URL
- `--offset N` — пропустить первые N URL
- `-v, --verbose` — отладочный вывод
- `-p, --pretty` — форматировать JSON красиво (каждый атрибут на новой строке)

## Прерывание и продолжение

Воркер можно прервать в любой момент (`Ctrl+C`). При следующем запуске
с теми же файлами он пропустит уже обработанные URL.

Чтобы начать заново, удалите файл прогресса:
```bash
rm books.progress.json
```

## Формат входного файла

Один URL на строку, пустые строки и строки, начинающиеся с `#`, игнорируются:

```
# Комментарий
https://ast.ru/book/zvezdy-iz-pepla-873454/
https://ast.ru/book/zvyozdy-eshchye-goryat-881439/
...
```

## Формат выходного файла

По умолчанию — компактный JSONL (одна книга на строку):

```json
{"title": "Звезды из пепла", "author": "Тория Дрим", "isbn": "978-5-17-157591-5", ...}
{"title": "Звёзды ещё горят", ...}
```

С флагом `--pretty` — многострочный JSON (каждый атрибут на новой строке, книги
разделены пустой строкой). Удобно для просмотра в редакторе, но файл
получается в ~2-3 раза больше:

```json
{
  "title": "Звезды из пепла",
  "author": "Тория Дрим",
  "isbn": "978-5-17-157591-5",
  "year": "2023",
  "pages": "480",
  ...
  "extras": {
    "бумага": "...",
    "ббк": "..."
  }
}

{
  "title": "Звёзды ещё горят",
  ...
}
```

Компактный JSONL удобно читать через `pandas`:
```python
import pandas as pd
df = pd.read_json("books.jsonl", lines=True)
```

## Производительность

- На MacBook Pro M4 Pro с 30 потоками — примерно 30-50 книг/сек.
- 100 000 URL ≈ 1-2 часа.
- 1 000 000 URL ≈ 10-20 часов.

При агрессивной конкуррентности (50+) можно получить рейт-лимит от сервера.
Воркер сам обрабатывает 429/503, увеличивая задержку.

## Защита от банов

- Реалистичные User-Agent (ротация)
- Случайные задержки между запросами
- Уважение к `Retry-After` от сервера
- Лимит одновременных соединений на хост
- Корректные HTTP-заголовки (Accept, Accept-Language)
